📈 Overfitting vs Underfitting
Overfitting (High Variance)
Underfitting (High Bias)
good train, bad val
bad train AND bad val
Fixes: Overfitting
  • Regularization (L1/L2), dropout, early stopping, more data
Fixes: Underfitting
  • More features, complex model, more epochs, more data
⚖️ Class Imbalance
imbalance data → SMOTE or Weighted loss fn → Penalize
  • SMOTE = synthetic minority over-sampling technique
  • Weighted loss = penalize minority misclassification more
  • Can also under-sample majority class
  • Dimensionality reduction does NOT fix imbalance
  • Accuracy is misleading on imbalanced data → use Recall/F1
🎛️ Regularization L1 vs L2
L1 Lasso
L2 Ridge
sparsity, feature selection
correlated features, shrinks all
  • L1 drives some coefficients exactly to zero → sparse model
  • L2 distributes penalty across all features smoothly
  • Both reduce overfitting by penalizing large weights
🎭 Bias-Variance Tradeoff
High Bias
High Variance
underfits, simple model
overfits, complex model
  • Goal: balance both — the sweet spot = generalization
  • More complexity → bias↓ but variance↑
  • Regularization + more data → shrinks variance without bias spike
📉 Loss Curves
  • train loss ↓ + val loss ↓ together = healthy training
  • train loss ↓ + val loss ↑ = overfitting
  • both high = underfitting
  • Monitored in SageMaker Debugger during training
  • Early stopping = halt when val loss stops improving

🌳 Boosting vs Bagging
Boosting (XGBoost)
Bagging (Random Forest)
sequential, corrects errors, bias+variance
parallel, independent, variance only
  • XGBoost = gradient boosting, each tree learns from previous errors
  • Random Forest = builds many independent trees, averages results
  • XGBoost often wins tabular competitions
📏 Evaluation Metrics
Precision
Recall
TP/(TP+FP) — of positives predicted, how many real?
TP/(TP+FN) — of all real positives, how many found?
  • AUC-ROC: overall model ranking ability
  • RMSE/MAE: regression errors (continuous output)
  • Use Recall when missing positives is costly (fraud, cancer)
  • Use Precision when false alarms are costly (spam filter)

🔲 Grid Search (HPO)
hyperparam space → Entire →
grid search
  • Grid Search = exhaustive, covers entire hyperparam space
  • Best when model trains fast AND search space is small
  • Random Search = samples, no full coverage guarantee
  • Grid Search is slow for large, high-dimensional spaces
  • Use Bayesian / Hyperband for large spaces
