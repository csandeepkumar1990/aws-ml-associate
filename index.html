📡Model Monitor & DriftNew Session
  • Data drift — input distribution changes over time → Model Monitor tracks it; retrain on latest data
  • Concept drift — relationship between input features and target changes (e.g., fraud patterns); causes accuracy drop despite no model change
  • Model drift — model's assumptions become outdated → periodic retrain + model versioning
  • Combined fix: continuous statistical monitoring with alerts + retrain on recent data
  • CloudWatch — monitor API call metrics on SageMaker endpoints; alarm → SNS notification on threshold breach
⚖️SageMaker ClarifyNew Session
  • CDD (Conditional Demographic Disparity) — disparity in positive prediction rates across demographic groups conditioned on a feature like income; detects bias invisible in overall outcomes
  • On-demand bias detection — analyzes captured inference data from deployed real-time endpoints
  • SHAP values — local explanation: contribution of each feature to a specific prediction instance
  • PDP (Partial Dependence Plots) — global explanation: marginal effect of feature across entire dataset
  • Use SHAP for individual loan/approval decisions; use PDP for overall model behavior understanding
  • ✗ Model Monitor = drift detection, NOT bias analysis
  • Feature importance — use Clarify to evaluate domain-specific features for credit risk improvement
📦SageMaker Model RegistryNew Session
  • Collections — group existing model groups into high-level categories (fraud, risk, segmentation); non-disruptive, does not move artifacts
  • Cross-account access — attach resource-based policy to Registry; grants other AWS accounts access
  • Model groups — manage versions + deployments with minimal operational overhead
  • Metadata captured: data sources, hyperparameters, training code, artifacts
  • Best for: versioning, repeatability, auditability in regulated industries (pharma, finance)
  • ✗ Tags — metadata only, not hierarchical structure for grouping
  • ✗ Experiments — tracks runs/trials; NOT a governance or approval tool
🚦Deployment StrategiesNew Session
  • Canary release — small % of traffic to new model fleet as safety guardrail; monitor then expand
  • Blue/Green — full parallel fleet; baking period; CloudWatch alarm triggers automatic rollback to blue
  • A/B Testing (Production Variants) — relative weight ratios (weight 20 ≠ 20% of traffic)
  • Shadow Testing — copy of live traffic sent to new model; responses logged only, NOT returned to caller
  • Auto-scaling (target tracking) — adjusts instance count based on CPU utilization or invocations per instance; maintains steady performance during spikes
  • Real-time hosting + auto-scaling — best for recommendation engine during peak shopping seasons
🚀Inference ModesNew Session
  • Real-Time — persistent endpoint, low-latency, auto-scaling; ≤6 MB payload, ≤60 sec; best for product recommendations
  • Serverless — auto-scales, pay-per-use; handles infrequent spikes; minimizes cost during low traffic; cold starts possible; ≤4 MB, ≤60 sec
  • Multi-Model Endpoint — multiple models on single instance; dynamic load/unload; cost savings for route optimization
  • Async Inference — ≤1 GB, up to 1 hr; scales to zero; NOT for strict real-time
  • Batch Transform — offline bulk from S3; NOT for real-time
  • Inference Recommender — load-tests across instance types → recommends cost/performance balance
🔁Training OptimizationNew Session
  • SageMaker Warm Pools — reuses provisioned training instances between back-to-back jobs; reduces startup time for iterative HPT runs
  • ✗ Managed Spot Training — cost savings only, not faster startup
  • ✗ Training Compiler — optimizes training ops, not instance startup time
  • Managed Spot Training — up to 90% cost reduction; S3 checkpoints for automatic retry on interruption
  • Oscillating loss → reduce learning rate; too high = overshooting loss minima, creates oscillation with SGD
  • Checkpoints to S3 — enables resume after Spot instance interruption
🎛️Hyperparameter Tuning (HPO)New Session
  • Bayesian Optimization — uses prior trial results; best for complex models with large search space; switch to this if Random Search trials are not converging
  • Hyperband — early-stops underperforming trials; fastest for large search spaces
  • Random Search — good when only a few hyperparameters are important
  • Grid Search — exhaustive, most computationally expensive
  • When trials don't converge: narrow the range of critical hyperparameters AND switch strategy to Bayesian

📐Algorithm SelectionNew Session
  • XGBoost — handles class imbalance via weighted classes + regularization; custom objective for precision & recall; best for fraud detection
  • Random Cut Forest (RCF) — unsupervised anomaly detection in sensor/IoT time-series; detects impending equipment failures
  • Linear Learner — handles class imbalance + feature interdependencies; built-in; LEAST operational effort for churn prediction
  • Logistic regression — binary classification (default/no-default)
  • K-Means — unsupervised clustering (customer segmentation) ≠ KNN
  • RNN — recurrent neural network; time-series forecasting (electricity demand)
  • PCA — dimensionality reduction; feature extraction
🎯Ensemble MethodsNew Session
  • Stacking — use predictions from logistic regression + decision tree + SVM as inputs to meta-model (random forest); best for combining diverse model types
  • Bagging — same model on different data subsets; reduces variance
  • AdaBoost — adaptive weight adjustment on misclassified samples
  • Gradient Boosting — sequential residual correction
  • XGBoost — regularized extreme gradient boosting; weighted classes; MOST SUITABLE for imbalanced fraud data
🧠Overfitting & RegularizationNew Session
  • Signs: model improves on validation in early epochs then degrades after certain epoch count
  • Dropout layers — randomly zero out neurons during training; prevents co-adaptation (overfitting)
  • Early stopping — halt training once validation loss stops improving; prevents further overfitting
  • Data augmentation — increase diversity of training data (rotations, flips, crops)
  • Ensembling — averaging predictions from multiple models reduces variance
  • Combined: dropout + early stopping = MOST LIKELY to mitigate neural network overfitting on medical images
📊Evaluation MetricsNew Session
  • MAE (Mean Absolute Error) — for continuous output like delivery hours; interpretable, robust to outliers
  • RMSE — penalizes large prediction errors more heavily
  • AUC ≈ 1.0 — excellent discriminatory power (defaulters vs non-defaulters)
  • AUC ≈ 0.5 — random/no discrimination power | AUC ≈ 0 — inverted predictions
  • ROC + AUC — best for binary classification with class imbalance (10% defaulters)
  • Precision + Recall — evaluate ability to find positives while controlling false positives/negatives
  • Confusion matrix — TP, FP, TN, FN; foundation for precision, recall, F1 calculation
⚗️Feature EngineeringNew Session

  • Standardization — building_size (continuous numerical feature)
  • Binning — group continuous values into discrete ranges
  • EDA first — understand distribution, missing values, class imbalance (5% disease positive) before deciding ML feasibility with only 1,000 records
  • Stratified split — preserves class distribution; critical for imbalanced data

🎯Recall vs Precision (PT-2)PT-2 · 7×
  • Recall — of all actual positives, how many caught? minimize false negatives
  • Precision — of predicted positives, how many correct? minimize false positives
  • Healthcare / fraud → prioritize Recall (missing = dangerous)
  • Loan denial → balance Precision + Recall
  • Imbalanced datasets: prefer AUC-ROC or F1 over accuracy
📉Overfitting: L1 / L2 / XGBoostPT-2 · 6×
  • L1 (Lasso) — shrinks some coefficients to zero → feature selection / elimination
  • L2 (Ridge) — shrinks all evenly, none to zero; smooths but keeps all features
  • XGBoost overfit fix — reduce max_depth (limits tree complexity)
  • Deep learning overfit fix: Dropout + Early Stopping
  • Use L1 when you want automatic irrelevant feature elimination

🌳Ensemble Methods Deep-Dive (PT-4)PT-4 · 4×
  • Boosting — sequential; each model corrects prior errors; reduces both bias AND variance; XGBoost/AdaBoost
  • Bagging — parallel independent models on bootstrap subsets; reduces variance only; Random Forest
  • Random Forest — efficient, handles correlated features; + feature selection + periodic retraining = predictive maintenance
  • L2 (Ridge) — correlated features (churn): shrink all coefficients evenly; none zeroed
  • L1 (Lasso) — sparse high-dimensional data: forces some weights to zero → feature selection
  • Cross-validation + model simplification = best fix for overfitting in recommendation systems
